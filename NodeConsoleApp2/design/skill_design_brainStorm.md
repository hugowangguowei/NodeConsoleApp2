# 护甲相关技能设计头脑风暴（Skill Design BrainStorm - Armor）

> 目的：整理“按部位护甲 + 回合制AP + 1v1 + 一回合可编排多个技能释放”的护甲技能设计思路，形成可持续的战斗循环与相互克制关系。
>
> 关键词：获取护甲、消耗护甲、护甲使用方式（部位/重分配/形态）、破甲与反破甲、编排连招（多技能序列）。

---

## 1. 核心设计目标（为什么护甲要“可玩”）

护甲不应仅作为“被动减伤数值”，而应同时承担：

1. **减伤层（基础规则）**：护甲 > 0 先扣护甲，再扣 HP。
2. **资源层（可消费）**：护甲可被主动消耗，转换为输出/节奏/免控/净化等收益。
3. **部位层（战术层）**：护甲分布在不同部位，可被针对与重分配，让“选部位”成为核心博弈。

> 形成循环的关键：护甲要能 **获得→分配/转化→消费换收益→被破坏→修复/再获得**。

---

## 2. 护甲获取方式（Gain Armor）

### 2.1 直接生成（稳健型）
- **定向加固**：选择一个部位/部位组加护甲（数值高、AP低），但覆盖面窄。
- **全身薄甲**：所有部位小额护甲，克制多段或多部位攻击，但对单点爆发抗性弱。
- **临时护甲（Temp Armor）**：持续 1 回合/2 回合，结束后衰减或清零；用于“预判爆发回合”。

### 2.2 进攻获得（进攻转防守）
- **吸甲（Armor Leech）**：伤害的一部分转化为自身护甲（可指定部位）。
- **偷甲（Steal Armor）**：命中某部位后偷取对方该部位护甲（强对抗性）。
- **破甲奖励（Break Reward）**：当你把敌方某部位护甲打到 0/破坏时，获得“护甲碎片/强化层”。

### 2.3 受击获得（挨打也有收益）
- **反应式镀层（Reactive Plating）**：某部位被命中后，下回合该部位获得护甲/修复提升。
- **护甲破裂阈值触发（Emergency Hardening）**：某部位首次被打穿触发一次护甲/减伤/免控。

### 2.4 以代价换护甲（高决策性）
- **HP 换护甲**：牺牲 HP 获取大量护甲（用于苟过关键回合）。
- **AP 换护甲**：本回合少输出换生存；形成节奏差。
- **弃甲换甲（Sacrifice & Reinforce）**：清空某些部位护甲，把部分转移到关键部位（有损耗）。

---

## 3. 护甲消耗方式（Spend Armor / Armor as Fuel）

### 3.1 护甲换输出（燃料型）
- **护甲灌注（Armor Infusion）**：消耗自身护甲，提升下一次攻击伤害/破甲/暴击/命中权重。
- **碎甲猛击（Shatter Strike）**：消耗自身护甲，提高本次破甲效率（高风险高收益）。
- **护甲爆裂（Armor Burst）**：主动清空某部位护甲，对敌造成等比例伤害或附加效果。

### 3.2 护甲换节奏（AP/速度/先手）
- **卸甲冲刺（Unarmored Dash）**：消耗护甲换速度（行动顺序提前）或返还少量 AP。
- **稳固姿态（Fortify Stance）**：以“本回合弱行动”换取下回合修复加成/额外资源。

### 3.3 护甲换生存功能（免控/净化/特殊吸收）
- **护甲完整性（Integrity）**：关键部位护甲 > 0 时抵消一次控制/负面效果；触发后扣护甲。
- **护甲净化（Armor Purge）**：消耗护甲移除 DOT / debuff（燃烧、流血、腐蚀）。

---

## 4. 护甲使用方式（按部位系统的“可玩性”来源）

### 4.1 护甲重分配（强建议作为体系核心）
- **护甲转移（Transfer）**：A 部位 → B 部位（建议存在 10%~30% 损耗，避免绝对最优解）。
- **护甲均摊（Equalize）**：把护甲平均化，避免被点杀薄弱部位。
- **护甲集中（Concentrate）**：将多部位护甲汇聚到关键部位，对抗单点爆发。

### 4.2 护甲形态（Armor Types / Layers）
为了减少“只加数值”的平庸感，可引入至少两种形态：
- **普通护甲（Armor）**：基础层，先扣护甲再扣 HP。
- **镀层/板甲（Plate）**：抗破甲强，但修复慢/修复成本高。
- **屏障（Barrier）**：吸收伤害但难修复；或仅吸收技能伤害/仅吸收普攻。

## 5. 对抗闭环：破甲与反破甲（Counter Loop）

> 完整循环必须允许：堆甲方能赢节奏、破甲方能拆体系、反破甲方能应对破甲。

### 5.1 破甲手段（Anti-Armor）
- **削减护甲上限（Reduce MaxArmor）**：让修复也变差（强克制）。
- **护甲腐蚀（Corrosion DOT on Armor）**：每回合掉护甲，逼迫对面花 AP 修复。
- **穿透/无视护甲（Penetration）**：建议做成“穿透 X 点/仅对某部位”而不是完全无视。
- **破坏状态（Broken）**：护甲=0 时进入破坏，禁止修复 1 回合或修复效率大幅下降。

### 5.2 反制破甲（Anti-Anti-Armor）
- **抗破甲（Fortified / Anti-Corrosion）**：降低被削上限/被偷/被腐蚀的比例。
- **替代层优先消耗**：例如屏障优先被消耗，保护护甲本体。
- **目标诱导/防局部爆发**：通过均摊/转移让对手的“锁部位连击”失效。

## 6. 与“多技能编排释放”结合的护甲流派循环模板

### 模板 A：重装修复流（稳健）
1) 生成全身薄甲/定向加固 → 2) 转移护甲补短板 → 3) 受击触发反应式修复 → 4) 稳固姿态攒资源 → 重复

克制点：腐蚀、削上限、破坏（禁止修复）。

### 模板 B：破甲爆发流（节奏）
1) 先上腐蚀/破坏 → 2) 锁定同一部位连击打穿 → 3) 护甲转输出打爆发 → 4) 对手修复时转火另一部位

克制点：均摊/转移、抗破甲、屏障层。

### 模板 C：护甲燃料连招流（技巧）
1) 集中加固堆高护甲 → 2) 卸甲冲刺抢先手 → 3) 护甲灌注输出 → 4) 护甲不足时吸甲/偷甲续航

克制点：强控打断、持续腐蚀、强破坏导致“护甲燃料断供”。

## 7. 可落地的护甲技能标签建议（用于平衡分析/工具统计）

建议为护甲相关技能定义标签集合（可用于 `skill_balance_tool` 的分布统计）：

- **功能类**：`armor_gain` / `armor_repair` / `armor_transfer` / `armor_equalize` / `armor_concentrate` / `armor_consume`
- **对抗类**：`anti_armor` / `anti_anti_armor` / `corrosion` / `break` / `max_armor_reduction` / `penetration`
- **形态类**：`armor_plate` / `armor_barrier`
- **节奏类**：`speed_gain` / `ap_refund` / `tempo`
- **部位类**：`part_specific` / `part_group` / `all_parts`

---

## 8. 实用小机制（提升“可玩性”但实现成本可控）

- **修复效率（repairEfficiency）**：不同角色/装备影响修复值倍率。
- **修复递减**：同一部位连续修复效率下降（防止无限单点最优）。
- **技能前置条件**：如“盾击要求躯干护甲>0”，“铁头槌要求头部护甲>0”。
- **部位数量=8 的兼容**：用部位标签组（head/torso/arm/leg/extra...）支持敌人特殊部位。
