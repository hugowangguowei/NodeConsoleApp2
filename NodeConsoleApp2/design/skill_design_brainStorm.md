# 技能设计头脑风暴（Skill Design BrainStorm）

> 目的：沉淀技能体系的“可复用设计模块”，支持后续持续扩展（护甲、防御、攻击、控制、治疗、召唤等）。
>
> 当前覆盖：**护甲/防御类技能** 与 **攻击类技能** 两大模块。

---

## 1. 总纲：通用设计目标与循环框架

### 1.1 核心设计目标（让技能体系“可循环”）

- **资源循环**：获得资源 → 分配/转化 → 消耗换收益 → 被反制/被破坏 → 再次获得。
- **对抗闭环**：每一条强路线都必须有明显克制点与反制手段（避免“单解”）。
- **部位策略**：在“按部位”规则下，技能必须能表达“选部位/改部位/惩罚部位”的策略价值。
- **编排价值**：适配“一回合多技能编排释放”，让 SETUP/PAYOFF 分工明确。

### 1.2 通用术语（建议在设计/数据/工具中统一使用）

- **SETUP**：制造窗口（破甲、上状态、调整资源/部位、限制对手）。
- **PAYOFF**：吃窗口收益（爆发、强控、资源拉开）。
- **Counter**：对抗工具（反制对手路线）。
- **Tempo**：节奏技能（速度/先手/AP 交换）。

### 1.3 通用标签模板（用于平衡分析/工具统计）

> 这里仅给“模板”，各技能域（护甲/攻击/控制等）再补充各自标签。

- 功能类：`set_up` / `payoff` / `counter` / `tempo`
- 价值维度：`resource_gain` / `resource_spend`
- 目标维度：`part_specific` / `part_group` / `all_parts`

## 2. 防御模块：护甲体系（Armor / Defensive)

> 护甲不应仅作为“被动减伤数值”，而应同时承担：
>
> 1) **减伤层（基础规则）**：护甲 > 0 先扣护甲，再扣 HP。
> 2) **资源层（可消费）**：护甲可被主动消耗，转换为输出/节奏/免控/净化等收益。
> 3) **部位层（战术层）**：护甲分布在不同部位，可被针对与重分配。
>
> 护甲循环关键：**获得→分配/转化→消费换收益→被破坏→修复/再获得**。

---

### 2.1 护甲获取方式（Gain Armor）

#### 2.1.1 直接生成（稳健型）
- **定向加固**：选择一个部位/部位组加护甲（数值高、AP低），但覆盖面窄。
- **全身薄甲**：所有部位小额护甲，克制多段或多部位攻击，但对单点爆发抗性弱。
- **临时护甲（Temp Armor）**：持续 1 回合/2 回合，结束后衰减或清零；用于“预判爆发回合”。

#### 2.1.2 进攻获得（进攻转防守）
- **吸甲（Armor Leech）**：伤害的一部分转化为自身护甲（可指定部位）。
- **偷甲（Steal Armor）**：命中某部位后偷取对方该部位护甲（强对抗性）。
- **破甲奖励（Break Reward）**：当你把敌方某部位护甲打到 0/破坏时，获得“护甲碎片/强化层”。

#### 2.1.3 受击获得（挨打也有收益）
- **反应式镀层（Reactive Plating）**：某部位被命中后，下回合该部位获得护甲/修复提升。
- **护甲破裂阈值触发（Emergency Hardening）**：某部位首次被打穿触发一次护甲/减伤/免控。

#### 2.1.4 以代价换护甲（高决策性）
- **HP 换护甲**：牺牲 HP 获取大量护甲（用于苟过关键回合）。
- **AP 换护甲**：本回合少输出换生存；形成节奏差。
- **弃甲换甲（Sacrifice & Reinforce）**：清空某些部位护甲，把部分转移到关键部位（有损耗）。

---

### 2.2 护甲消耗方式（Spend Armor / Armor as Fuel）

#### 2.2.1 护甲换输出（燃料型）
- **护甲灌注（Armor Infusion）**：消耗自身护甲，提升下一次攻击伤害/破甲/暴击/命中权重。
- **碎甲猛击（Shatter Strike）**：消耗自身护甲，提高本次破甲效率（高风险高收益）。
- **护甲爆裂（Armor Burst）**：主动清空某部位护甲，对敌造成等比例伤害或附加效果。

#### 2.2.2 护甲换节奏（AP/速度/先手）
- **卸甲冲刺（Unarmored Dash）**：消耗护甲换速度（行动顺序提前）或返还少量 AP。
- **稳固姿态（Fortify Stance）**：以“本回合弱行动”换取下回合修复加成/额外资源。

#### 2.2.3 护甲换生存功能（免控/净化/特殊吸收）
- **护甲完整性（Integrity）**：关键部位护甲 > 0 时抵消一次控制/负面效果；触发后扣护甲。
- **护甲净化（Armor Purge）**：消耗护甲移除 DOT / debuff（燃烧、流血、腐蚀）。

---

### 2.3 护甲使用方式（按部位系统的“可玩性”来源）

#### 2.3.1 护甲重分配（强建议作为体系核心）
- **护甲转移（Transfer）**：A 部位 → B 部位（建议存在 10%~30% 损耗，避免绝对最优解）。
- **护甲均摊（Equalize）**：把护甲平均化，避免被点杀薄弱部位。
- **护甲集中（Concentrate）**：将多部位护甲汇聚到关键部位，对抗单点爆发。

#### 2.3.2 护甲形态（Armor Types / Layers）
为了减少“只加数值”的平庸感，可引入至少两种形态：
- **普通护甲（Armor）**：基础层，先扣护甲再扣 HP。
- **镀层/板甲（Plate）**：抗破甲强，但修复慢/修复成本高。
- **屏障（Barrier）**：吸收伤害但难修复；或仅吸收技能伤害/仅吸收普攻。

---

### 2.4 对抗闭环：破甲与反破甲（Counter Loop）

> 完整循环必须允许：堆甲方能赢节奏、破甲方能拆体系、反破甲方能应对破甲。

#### 2.4.1 破甲手段（Anti-Armor）
- **削减护甲上限（Reduce MaxArmor）**：让修复也变差（强克制）。
- **护甲腐蚀（Corrosion DOT on Armor）**：每回合掉护甲，逼迫对面花 AP 修复。
- **穿透/无视护甲（Penetration）**：建议做成“穿透 X 点/仅对某部位”而不是完全无视。
- **破坏状态（Broken）**：护甲=0 时进入破坏，禁止修复 1 回合或修复效率大幅下降。

#### 2.4.2 反制破甲（Anti-Anti-Armor）
- **抗破甲（Fortified / Anti-Corrosion）**：降低被削上限/被偷/被腐蚀的比例。
- **替代层优先消耗**：例如屏障优先被消耗，保护护甲本体。
- **目标诱导/防局部爆发**：通过均摊/转移让对手的“锁部位连击”失效。

---

### 2.5 与“多技能编排释放”结合的护甲流派循环模板

#### 2.5.1 模板 A：重装修复流（稳健）
1) 生成全身薄甲/定向加固 → 2) 转移护甲补短板 → 3) 受击触发反应式修复 → 4) 稳固姿态攒资源 → 重复

克制点：腐蚀、削上限、破坏（禁止修复）。

#### 2.5.2 模板 B：破甲爆发流（节奏）
1) 先上腐蚀/破坏 → 2) 锁定同一部位连击打穿 → 3) 护甲转输出打爆发 → 4) 对手修复时转火另一部位

克制点：均摊/转移、抗破甲、屏障层。

#### 2.5.3 模板 C：护甲燃料连招流（技巧）
1) 集中加固堆高护甲 → 2) 卸甲冲刺抢先手 → 3) 护甲灌注输出 → 4) 护甲不足时吸甲/偷甲续航

克制点：强控打断、持续腐蚀、强破坏导致“护甲燃料断供”。

---

### 2.6 护甲技能标签建议（用于平衡分析/工具统计）

建议为护甲相关技能定义标签集合（可用于 `skill_balance_tool` 的分布统计）：

- **功能类**：`armor_gain` / `armor_repair` / `armor_transfer` / `armor_equalize` / `armor_concentrate` / `armor_consume`
- **对抗类**：`anti_armor` / `anti_anti_armor` / `corrosion` / `break` / `max_armor_reduction` / `penetration`
- **形态类**：`armor_plate` / `armor_barrier`
- **节奏类**：`speed_gain` / `ap_refund` / `tempo`
- **部位类**：`part_specific` / `part_group` / `all_parts`

---

### 2.7 实用小机制（提升“可玩性”但实现成本可控）

- **修复效率（repairEfficiency）**：不同角色/装备影响修复值倍率。
- **修复递减**：同一部位连续修复效率下降（防止无限单点最优）。
- **技能前置条件**：如“盾击要求躯干护甲>0”，“铁头槌要求头部护甲>0”。
- **部位数量=8 的兼容**：用部位标签组（head/torso/arm/leg/extra...）支持敌人特殊部位。

## 3. 进攻模块：攻击技能（Offensive)

### 3.1 攻击技能方案（围绕护甲体系的一套可循环攻击技能）

> 目标：提供一套“进攻侧技能包”，能与本文件的护甲循环形成稳定对抗：
>
> - 能打护甲（破甲/腐蚀/削上限/破坏）
> - 能绕护甲（穿透/转化伤害类型）
> - 能利用护甲（偷甲/吸甲/以护甲作为燃料）
> - 能形成连招（SETUP -> PAYOFF），适配“一回合多技能编排释放”

#### 3.1.1 设计原则

1) **攻击技能必须具备“目标部位策略”**：
   - 同一技能的价值取决于目标部位当前护甲厚度、是否处于 `Broken`、是否存在 `Plate/Barrier` 等形态。

2) **SETUP / PAYOFF 明确分工**：
   - SETUP：降低护甲、制造破坏窗口、打乱护甲分布。
   - PAYOFF：把窗口转化为 HP 伤害/控制/资源优势。

3) **避免“单技能万能”**：
   - 破甲强的技能，在穿透/HP 直伤方面要弱；
   - 穿透强的技能，成本高或对护甲类目标收益有限；
   - 让玩家必须在同回合编排多技能来达成爆发。

#### 3.1.2 技能分类（建议 6 类）

#### A. 破甲打击（Armor Damage / Shatter）
用于把某部位护甲快速打到 0，制造 Broken 窗口。

- **技能 A1：碎甲斩（Shatter Slash）**
  - 概念：对单一部位造成高额护甲伤害（优先扣护甲，不溢出到 HP）。
  - 标签建议：`DMG_ARMOR` `ABS` `INSTANT` `ONE_SHOT` `anti_armor` `part_specific`
  - 玩法定位：核心破甲工具。

- **技能 A2：连环凿击（Chipping Combo）**
  - 概念：同一回合若连续命中同部位，则后续段对护甲伤害递增（鼓励“锁部位连击”）。
  - 风险：容易过强，建议递增幅度有限或需要更高 AP。
  - 标签建议：`COMBO` `SETUP`

#### B. 护甲腐蚀（Armor Corrosion / DOT）
用于对抗“修复流/堆甲流”，逼迫对方花 AP 修复。

- **技能 B1：酸蚀印记（Corrosive Mark）**
  - 概念：给目标某部位施加护甲腐蚀（每回合掉护甲），持续 2~3 回合。
  - 标签建议：`corrosion` `DELAYED` `MULTI_TURN` `anti_armor`

- **技能 B2：锈蚀扩散（Rust Spread）**
  - 概念：若目标存在腐蚀效果，则将其复制到另一个随机部位（或相邻部位组）。
  - 玩法：破坏对手“集中加固”的价值。

#### C. 破坏窗口（Broken Window / Repair Lock）
让“护甲打穿”产生大收益：限制修复、降低转移效率等。

- **技能 C1：毁甲钉刺（Armor Breaker Spike）**
  - 概念：若目标部位护甲被打到 0，则施加 `Broken`：下回合禁止修复或修复效率 -X%。
  - 玩法：将“打穿护甲”从一次性变成节奏优势。
  - 标签建议：`break` `COND_TARGET_ARMOR_BROKEN`

- **技能 C2：断裂追击（Crack Pursuit）**
  - 概念：若目标部位处于 `Broken`，则对 HP 造成额外伤害（PAYOFF）。
  - 标签建议：`PAYOFF` `DMG_HP` `COND_TARGET_ARMOR_BROKEN`

#### D. 穿透/绕甲（Penetration / Convert）
不追求把护甲打穿，而是绕过部分护甲保证输出下限。

- **技能 D1：穿甲突刺（Piercing Thrust）**
  - 概念：对单部位造成伤害，其中 X 点直接作用于 HP（或“先扣一定量护甲，再把剩余穿透到 HP”）。
  - 价值：对厚甲目标也能稳定压血线。
  - 标签建议：`penetration` `DMG_HP` `DMG_ARMOR`

- **技能 D2：弱点剖解（Expose Weakness）**
  - 概念：本回合对同一部位的下一次攻击获得穿透（作为连招增益）。
  - 玩法：更偏 SETUP，配合其它技能一起编排。

#### E. 偷甲/吸甲（Armor Steal / Leech）
把对手的护甲“变成你的资源”，让进攻带防守。

- **技能 E1：掠甲斩（Armor Reave）**
  - 概念：对目标部位造成护甲伤害，并将其中一部分转移到自身某部位护甲。
  - 玩法：对抗修复流，同时自我续航。
  - 标签建议：`armor_gain` `anti_armor` `tempo`

- **技能 E2：吸甲回旋（Leeching Sweep）**
  - 概念：对多个部位造成少量护甲伤害，并按总伤害转化为自身“屏障/临时护甲”。
  - 注意：在 1v1 的 AOE 定义下，“多部位”是合理的 AOE。

#### F. 护甲燃料爆发（Armor as Fuel - Offensive）
让攻击者自己也需要“管理护甲资源”，形成高风险高收益打法。

- **技能 F1：崩甲怒击（Armor-Driven Smash）**
  - 概念：消耗自身某部位护甲，提升本次攻击破甲与 HP 伤害（护甲越高越强）。
  - 玩法：鼓励先堆甲再爆发，形成自带节奏。

- **技能 F2：卸甲疾袭（Unarmored Assault）**
  - 概念：若自身某部位护甲低/为 0，则获得速度提升与额外穿透（“越脆越快”）。
  - 玩法：反向机制，适合刺客型 build。

#### 3.1.3 一回合多技能编排示例（可直接当关卡/敌人 AI 的行动模板）

#### Combo 1：标准破甲爆发（SETUP -> PAYOFF）
1) `碎甲斩`（锁定胸/躯干，尽量把护甲打到 0）
2) `毁甲钉刺`（若护甲=0，施加 Broken 禁修复）
3) `断裂追击`（对 Broken 部位追加 HP 伤害）

#### Combo 2：对抗修复流（持续施压）
1) `酸蚀印记`（给目标主防御部位上腐蚀）
2) `锈蚀扩散`（打散对手集中加固）
3) `穿甲突刺`（稳定压血线，避免对方只修甲不掉血）

#### Combo 3：掠夺续航流（进攻带防守）
1) `掠甲斩`（偷甲给自己的关键部位）
2) `连环凿击`（锁部位扩大破甲）
3) 视对方修复情况接 `穿甲突刺` 或 `断裂追击`

#### 3.1.4 与标签体系的对应建议

为了方便后续 `skill_balance_tool` 统计“护甲相关攻击技能”的分布，建议至少保证：

- 破甲类（A/C）：`anti_armor` + `break` 覆盖充分
- 绕甲类（D）：`penetration` 至少有 2~3 个不同成本/倍率的技能
- 反制修复（B）：`corrosion` 至少存在 2 种（单点 + 扩散/转移惩罚）
- 可产生资源循环（E/F）：至少 1 种“偷甲/吸甲”，至少 1 种“护甲燃料爆发”

## 4. 扩展模块占位（后续其它技能域建议按相同模板补充）

> 未来技能域示例：控制（Control）、治疗（Healing）、召唤（Summon）、陷阱/地形（Hazard）、资源系统（Resource）等。

### 4.X 模块模板（建议套用）

1. **模块目标**：该技能域解决什么问题？克制什么？被谁克制？
2. **核心资源与循环**：该模块的“获得/消耗/转化/反制”闭环是什么？
3. **技能分类（建议 4~8 类）**：每类给 2~4 个技能点子。
4. **同回合编排示例**：至少 2 套 SETUP->PAYOFF 与 1 套对抗连招。
5. **标签建议**：保证能被平衡工具统计覆盖。
